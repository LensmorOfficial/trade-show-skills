# 展会全生命周期实战指南

如何串联 6 个技能，从选展到成单——完整流程演示。

> **说明**：本文中的公司和联系人均为虚构，仅用于示例目的。

---

## 场景设定

**公司**：Vantara Systems — 一家向食品饮料制造商销售生产排程 SaaS 的 B2B 公司。聚焦中型市场，平均合同 8 万–20 万欧元，销售周期 6–9 个月。

**参展团队**：3 人——1 名销售负责人、1 名方案工程师、1 名市场经理。

**目标**：获取 8–12 条合格线索（A/B 级），识别竞品定位变化，并在展会上验证正在内测的新产线平衡模块的产品-市场契合度。

**目标展会**：Anuga FoodTec（科隆）——两年一届，约 1,700 家参展商，聚焦食品加工技术采购商。

---

## 第一阶段：展前（开展前 8–12 周）

### 第一步：用 `trade-show-finder` 评估展会

团队还不确定今年选 Anuga FoodTec 还是 ProPak Europe。提问：

```
比较一下 Anuga FoodTec 和 ProPak Europe，我们卖生产排程 SaaS 给食品制造商，聚焦 DACH 和 Benelux 市场。
```

**输出**：两个展会的对比表格，含日期、场馆、参展商/观众数量和受众画像。技能推荐 Anuga FoodTec——工厂运营和生产管理层观众比例更高，DACH 覆盖更成熟，且有独立的软件专区。

**传递给下一步的信息**：展会名称、日期、访客画像、参展商数量——这些直接进入预算模型。

---

### 第二步：用 `trade-show-budget-planner` 建立 ROI 案例

确认目标展会后，市场经理需要向 CFO 说明这笔钱值不值得花：

```
帮我制定 Anuga FoodTec 的参展预算：20 平米展位、3 人团队、5 天展期。我们卖生产排程 SaaS，平均合同 12 万欧元，帮我建一个 ROI 案例。
```

**输出**：展位/基建、差旅、营销物料、运营费用的逐项预算。包含保守/基准/乐观三种情景的 ROI 预测。盈亏平衡计算：成交 1.4 单即可回本。行业线索获取成本基准数据。

**传递给下一步的信息**：CFO 审批用的备忘录。"目标线索数（8–12 条）"和"合同价值（12 万欧）"成为此行的成功指标，也决定了在展位上主动推进会议邀约的力度。

---

### 第三步：用 `booth-invitation-writer` 预约展位流量

展前六周，销售负责人手上有 80 个联系人——包括现有客户、沟通过的潜在客户和 DACH 食品制造商冷名单。需要三封不同的邀请邮件：

```
帮我写 Anuga FoodTec 的展位邀请邮件，3 类受众：
1. 现有客户（7 家）——邀请他们预约 30 分钟产品更新沟通
2. 温潜客（25 个联系人）——过去 6 个月内沟通过的工厂经理和运营总监
3. 冷外联（48 个联系人）——DACH 食品制造商，尚未联系过

我们在 5.1 馆 C42 展位，展会期间将发布内测中的新产线平衡模块。
```

**输出**：三套针对不同受众的邮件模板，各有不同语气、切入角度和行动号召。冷邮件序列附邮件主题行 A/B 测试版本。展前 14 天自动发送未读提醒邮件。

**传递给下一步的信息**：展前已预约 12 个会议。这些预约进入展会现场日程。温名单和冷名单邮件间隔 5 天发出；提醒邮件在展前 10 天发出。

---

## 第二阶段：展会现场（展期期间）

### 第四步：用 `badge-qualifier` 实时定性线索

展会第一天结束：销售负责人共有 14 次接触——3 次实质对话、7 次简短交流、4 次只扫了 badge 没有对话。当晚在酒店，他处理当天的记录：

```
帮我定性 Anuga FoodTec 第一天的线索。我们卖生产排程 SaaS。ICP：食品制造商工厂总监和运营经理，200 人以上，DACH 地区。

1. Petra Zimmermann，Ritter Sport 运营总监——25 分钟深入对话。目前用电子表格排程，对多产线之间的手工协调非常不满。提到"今年有批准预算，旺季前需要上线"。给了名片，要求我们发提案。

2. Badge 扫描：Marco Bianchi，百味来集团工艺工程师。没有对话，拿了我们的宣传册。

3. Tom van der Berg，FrieslandCampina 运营副总裁——10 分钟交流。有兴趣，但说"刚完成大型 ERP 上线，现在可能不是时候"。还是拿了我们的名片。

4. Ingrid Paulsen，Arla Foods 采购经理——演示了产线平衡模块，询问了 3 个工厂的报价。说 IT 部门需要介入，没有给出时间表。
```

**输出**：四张线索卡片，含分级、决策权评估和明确的下一步建议。Petra 评为 Hot（决策权 + 明确需求 + 预算已批准 + 同意接收提案）。Ingrid 评为 Warm（真实兴趣 + 价格讨论，但缺少时间表和 IT 决策参与）。Tom 评为 Cold（真实对话，但明确表示时机未到）。Marco 评为 Cold（仅 Badge 扫描）。

**关键价值**：技能明确指出 Marco 的 Cold 评级不应因"百味来"的品牌知名度而被拔高。Tom 的"时机不对"是真实反馈，而非含蓄的 Yes——展后立即安排 demo 属于过度跟进。

**传递给下一步的信息**：分级结果（1 Hot、1 Warm、2 Cold）直接映射到第三阶段的跟进序列。

---

### 第五步：用 `trade-show-competitor-radar` 记录竞品情报

方案工程师在 5.1 馆花 45 分钟走访竞品展台，记录笔记：

```
Anuga FoodTec 第二天竞品展台观察：

1. Preactor（西门子）——大展台，200 平米以上。新产品叫 "Opcenter Scheduling 2026"，展板上写着"AI 驱动的有限产能排程——减少 60% 的计划时间"。人流量大，多为工艺工程师。拿到了他们的产品单页。

2. Asprova——中等规模展台，常规定位，没有新东西。一位销售代表对一组访客说"我们单厂授权起步约 4 万欧"（偶然听到，未经证实）。

3. Logility——小展台，主要推需求计划方向，听下来不像是直接针对我们的客群。
```

**输出**：三份竞品情报笔记。Preactor/Opcenter 2026 被评为高威胁——一个直接攻击我方核心价值主张的量化声明（"减少 60% 计划时间"），辅以高人流量的观察支撑。Asprova 的报价（4 万欧）被打上 [HEARD] 标签，明确标注为未经证实。Logility 评为低威胁。

**来源标签的价值**：这份报告会传到后方销售团队。如果 Asprova 的价格以事实形式出现，可能会影响报价谈判。正确打标签后，它是一个"待核查的信号"，而不是可以直接引用的谈资。

**传递给下一步的信息**：Opcenter 2026 的声明进入展后产品团队复盘。行动建议（下载 Preactor 单页、准备应对"60% 计划时间"说法）记录在案，供展后一周内跟进。

---

## 第三阶段：展后（落地后 48 小时内）

### 第六步：用 `post-show-followup` 发出分级跟进邮件

销售负责人周二晚间落地。周三早上打开电脑前，手边已有 badge-qualifier 的清晰分级结果：

```
帮我写 Anuga FoodTec 展后跟进邮件序列。我们卖生产排程 SaaS。

Hot 线索：Petra Zimmermann，Ritter Sport 运营总监——要求提案，今年预算已批，旺季前需要上线。我们约好我发提案提纲并安排通话。

Warm 线索：Ingrid Paulsen，Arla Foods 采购经理——演示了产品，询问 3 个工厂报价，IT 需要介入但没有时间表。

2 条 Cold 线索：Marco Bianchi（仅 badge 扫描，无对话）和 Tom van der Berg（真实对话，但明确表示时机未到）。
```

**输出**：分级邮件序列。Petra 收到当天邮件，引用她的具体痛点（旺季前的排程协调故障），附上约定的提案提纲和日历预约链接。Ingrid 收到第二天邮件，用多工厂部署 ROI 框架切入，软性邀请安排一次包含 IT 负责人的通话。Marco 和 Tom 收到简短的低压力冷序列——不会假装发生过深度对话。

**48 小时窗口的价值**：Petra 这周会收到 30–50 封展会跟进邮件。周三到达的邮件——她还在科隆或在火车上——会被打开。周五到达的是收件箱噪音。而周三能发出个性化邮件，依赖的正是展会现场实时记录的 badge-qualifier 输出。

## 端到端流程总览

```
展前 8–12 周
  trade-show-finder
    → 比较展会，确认 Anuga FoodTec 为目标
    ↓
  trade-show-budget-planner
    → ROI 模型，CFO 审批备忘录，目标线索数：8–12 条
    ↓
  booth-invitation-writer
    → 三类受众邮件，展前预约 12 个会议

展会现场
    ↓
  badge-qualifier（实时或每日整理）
    → 分级结果：1 Hot、1 Warm、2 Cold——含明确下一步
    ↓
  trade-show-competitor-radar（每日记录）
    → Opcenter 2026 标记为高威胁；Asprova 报价标记为 [HEARD]

落地后 48 小时
    ↓
  post-show-followup
    → Petra：当天发提案 + 日历链接
    → Ingrid：第二天，多工厂 ROI 框架
    → Marco + Tom：冷序列，不虚构紧迫感
```

---

## 这套流程为什么有效

**关键在于交接点。** 每个技能的输出，下一个技能可以直接使用：

- `trade-show-finder` → 展会名称 + 访客画像 → 进入预算模型
- `trade-show-budget-planner` → 目标线索数 + 合同价值 → 成为现场团队的成功指标
- `booth-invitation-writer` → 预约会议 → 减少展位上的被动等待
- `badge-qualifier` → 含具体上下文的分级结果 → 实现当天发出个性化跟进
- `trade-show-competitor-radar` → 有来源标签的情报笔记 → 产品和销售团队拿到可信赖的信息
- `post-show-followup` → 个性化序列 → 在联系人还记得你的时候进入收件箱

**缺少这套工作流时常见的问题：**

- 不核验受众就决定参展 → 去错展会
- 没有展前预约会议 → 90% 时间被动等待
- 没有现场记录 → 展后只能发"很高兴认识你"的通用邮件
- 没有线索分级 → Petra 和 Marco 收到同一封邮件
- 竞品情报没有来源标签 → 销售团队把偷听到的价格当事实引用
